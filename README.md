# 智能测试用例生成工具

## 项目概述

这是一个基于人工智能的测试用例生成工具，能够根据需求文档自动生成功能测试用例和接口测试用例。该工具使用 PyQt5 构建图形用户界面，支持多种文档格式（Word、PDF、Excel、Markdown、TXT、JSON、YAML）的读取和处理，并利用大语言模型（如 DeepSeek）生成高质量的测试用例。

## 主要功能

1. **多格式文档支持**：支持读取 Word、PDF、Excel、Markdown、TXT、JSON、YAML 等格式的文档
2. **智能内容提取**：能够从文档中提取关键内容，包括文本、表格和图片
3. **OCR 文字识别**：使用 PaddleOCR 对图片中的文字进行识别
4. **多种测试用例设计方法**：支持等价类划分、边界值分析、决策表、状态转换、错误推测、场景法、因果图测试、正交分析法等多种测试用例设计方法
5. **行业特定优化**：针对不同行业（互联网/电子商务、保险、金融科技、医疗健康等）进行优化
6. **多种导出格式**：支持将生成的测试用例导出为 Word、TXT、Markdown、JSON、Excel 等格式

## 使用方法

1. **启动应用**：运行 `ai_assistant_tester_v1.1.0.py` 文件
2. **添加知识库**：点击"添加知识库"按钮，选择包含需求文档的目录
3. **选择文档**：在文件列表中选择需要处理的文档
4. **设置参数**：
   - 选择行业
   - 选择提示词模式（文档/参数输入）
   - 选择功能模式（功能测试用例/接口测试用例）
   - 选择用例设计方法（可多选）
5. **生成测试用例**：点击"开始推理"按钮，等待生成结果
6. **导出结果**：选择导出格式，点击"导出结果"按钮

## 配置说明

API 配置存储在 `api_config.json` 文件中，包括：
- `api_key`：API 密钥
- `base_url`：API 基础 URL
- `model`：使用的模型名称

## 依赖项

项目依赖以下 Python 库：

- tabulate：表格格式化

# 测试用例模板管理系统

这是一个用于管理测试用例模板的系统，支持模板的创建、编辑、导入导出等功能。

## 功能特点

1. 模板分类管理
   - 支持多种测试类型（功能测试、接口测试、性能测试、安全测试等）
   - 可自定义分类和描述

2. 模板管理
   - 创建新模板
   - 编辑现有模板
   - 删除模板
   - 导入/导出模板

3. 模板结构
   - 支持自定义模板结构
   - JSON格式存储
   - 支持版本控制

4. 用户界面
   - 直观的图形界面
   - 模板列表视图
   - 模板编辑界面
   - 实时预览

## 安装说明

1. 确保已安装Python 3.6+
2. 安装依赖包：
   ```bash
   pip install PyQt5
   ```

## 使用说明

### 启动程序

```bash
python template_manager_gui.py
```

### 基本操作

1. 查看模板
   - 在顶部下拉框选择分类
   - 模板列表会显示该分类下的所有模板

2. 添加模板
   - 点击"添加模板"按钮
   - 填写模板信息：
     - 模板ID：唯一标识符
     - 模板名称：显示名称
     - 分类：选择模板所属分类
     - 版本：模板版本号
     - 结构：JSON格式的模板结构

3. 编辑模板
   - 在列表中选择要编辑的模板
   - 点击"编辑模板"按钮
   - 修改相关信息
   - 点击"保存"保存修改

4. 删除模板
   - 在列表中选择要删除的模板
   - 点击"删除模板"按钮
   - 确认删除

5. 导入/导出模板
   - 点击"导入模板"按钮选择JSON文件导入
   - 点击"导出模板"按钮将当前模板导出为JSON文件

### 模板结构说明

模板使用JSON格式定义，基本结构如下：

```json
{
    "id": "模板ID",
    "name": "模板名称",
    "category": "分类ID",
    "version": "版本号",
    "structure": {
        "field1": "类型1",
        "field2": ["类型2", "类型3"],
        "field3": {
            "subfield1": "类型4",
            "subfield2": "类型5"
        }
    }
}
```

### 示例模板

1. 功能测试模板
```json
{
    "id": "func_basic",
    "name": "基础功能测试模板",
    "category": "functional",
    "version": "1.0",
    "structure": {
        "case_id": "<模块>_<3位序号>",
        "title": "<功能点>_<测试场景>",
        "priority": ["P0", "P1", "P2", "P3"],
        "precondition": "测试前置条件",
        "steps": ["步骤1", "步骤2", "步骤3"],
        "expected_result": "预期结果",
        "actual_result": "实际结果",
        "status": ["通过", "失败", "阻塞", "未执行"]
    }
}
```

2. 接口测试模板
```json
{
    "id": "api_basic",
    "name": "基础接口测试模板",
    "category": "interface",
    "version": "1.0",
    "structure": {
        "case_id": "API_<3位序号>",
        "api_name": "接口名称",
        "method": ["GET", "POST", "PUT", "DELETE"],
        "url": "接口URL",
        "headers": {},
        "request_body": {},
        "expected_status": 200,
        "expected_response": {},
        "validation_rules": []
    }
}
```

## 注意事项

1. 模板ID必须唯一
2. 模板结构必须符合JSON格式
3. 导入的模板文件必须是有效的JSON格式
4. 建议定期备份模板配置文件

## 常见问题

1. Q: 如何修改模板分类？
   A: 直接编辑template_config.json文件中的categories部分

2. Q: 如何添加新的模板字段？
   A: 在模板结构的structure部分添加新的字段定义

3. Q: 如何升级模板版本？
   A: 编辑模板时修改version字段，建议遵循语义化版本规范
